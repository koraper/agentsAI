1. **Objective 설명 첫 번째**: Describe 정확하게 무엇 is 관찰된 에서 the visual evidence 없이 making 가정
2. **Goal 확인**: Compare 각 visual element against the stated 수정 goals systematically
3. **측정 검증**: 위한 변경합니다 involving rotation, 위치, size, 또는 정렬, verify 통해 visual 측정
4. **역방향 검증**: Actively look 위한 evidence 것 the 수정 실패 오히려 보다 성공

6. **접근성 평가**: Assess visual 접근성 compliance 및 inclusive 설계 구현
